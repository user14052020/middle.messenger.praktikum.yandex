Для каждого спринта вы будете создавать отдельную ветку, в которой будете вести разработку (sprint_i, где i — номер спринта), открывать из неё pull request ветку main, а после того, как вашу работу проверил и принял ревьюер, мёрджить этот PR. Ветка deploy будет использоваться для публикации проекта.

git clone git@github.com:user14052020/sprint_1.git
cd sprint_1
git add .
git commit -m "Добавили свой index.html"
git push -u origin sprint_1

выбор ветки
$ git branch iss53
$ git checkout iss53
